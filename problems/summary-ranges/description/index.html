228. Summary Ranges

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

You are given a sorted unique integer array nums.

A range [a,b] is the set of all integers from a to b (inclusive).

Return the smallest sorted list of ranges that cover all the numbers in the array exactly. That is, each element of nums is covered by exactly one of the ranges, and there are no integers in the ranges that are not in nums.

Each range [a,b] in the list should be output as:

  • "a->b" if a != b
  • "a" if a == b

Example 1:

Input: nums = [0,1,2,4,5,7]
Output: ["0->2","4->5","7"]
Explanation: The ranges are:
[0,2] --> "0->2"
[4,5] --> "4->5"
[7,7] --> "7"

Example 2:

Input: nums = [0,2,3,4,6,8,9]
Output: ["0","2->4","6","8->9"]
Explanation: The ranges are:
[0,0] --> "0"
[2,4] --> "2->4"
[6,6] --> "6"
[8,9] --> "8->9"

Constraints:

  • 0 <= nums.length <= 20
  • -2³¹ <= nums[i] <= 2³¹ - 1
  • All the values of nums are unique
  • nums is sorted in ascending order

Solution

Python Solution

class Solution:
    def summaryRanges(self, nums: List[int]) -> List[str]:
        if not nums:
            return []
            
        ranges = []
        start = nums[0]
        
        for i in range(1, len(nums)):
            if nums[i] != nums[i-1] + 1:
                # End of current range
                if start == nums[i-1]:
                    ranges.append(str(start))
                else:
                    ranges.append(f"{start}->{nums[i-1]}")
                start = nums[i]
        
        # Handle the last range
        if start == nums[-1]:
            ranges.append(str(start))
        else:
            ranges.append(f"{start}->{nums[-1]}")
            
        return ranges

Time Complexity: O(n)

Where n is the length of the input array. We traverse the array once.

Space Complexity: O(1)

Not counting the space used for output, we only use a constant amount of extra space.

class Solution {
    public List summaryRanges(int[] nums) {
        List ranges = new ArrayList<>();
        if (nums == null || nums.length == 0) {
            return ranges;
        }
        
        int start = nums[0];
        
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[i-1] + 1) {
                // End of current range
                if (start == nums[i-1]) {
                    ranges.add(String.valueOf(start));
                } else {
                    ranges.add(start + "->" + nums[i-1]);
                }
                start = nums[i];
            }
        }
        
        // Handle the last range
        if (start == nums[nums.length-1]) {
            ranges.add(String.valueOf(start));
        } else {
            ranges.add(start + "->" + nums[nums.length-1]);
        }
        
        return ranges;
    }
}

JavaScript Solution

/**
 * @param {number[]} nums
 * @return {string[]}
 */
var summaryRanges = function(nums) {
    if (!nums.length) {
        return [];
    }
    
    const ranges = [];
    let start = nums[0];
    
    for (let i = 1; i < nums.length; i++) {
        if (nums[i] !== nums[i-1] + 1) {
            // End of current range
            if (start === nums[i-1]) {
                ranges.push(start.toString());
            } else {
                ranges.push(`${start}->${nums[i-1]}`);
            }
            start = nums[i];
        }
    }
    
    // Handle the last range
    if (start === nums[nums.length-1]) {
        ranges.push(start.toString());
    } else {
        ranges.push(`${start}->${nums[nums.length-1]}`);
    }
    
    return ranges;
};

Approach Explanation

The solution uses a single pass through the array to identify and format ranges. Here's how it works:

  1. Initialization:
    • Handle empty array case
    • Initialize start with first number
    • Prepare result list for ranges
  2. Main Process:
    • Iterate through array starting from second element
    • Check if current number is consecutive to previous
    • If not consecutive, end current range and start new one
    • Format range based on whether it's single number or range
  3. Final Step:
    • Handle the last range after loop ends
    • Format appropriately based on range size

Key points about the solution:

  • Takes advantage of the fact that array is sorted
  • Handles single-number ranges differently from multi-number ranges
  • Special handling for the last range is necessary
  • String formatting varies by language but logic remains same
