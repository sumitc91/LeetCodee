215. Kth Largest Element in an Array

Medium

Problem Description

Given an integer array nums and an integer k, return the kth largest element in the array.

Note that it is the kth largest element in the sorted order, not the kth distinct element.

Can you solve it without sorting?

Examples

Example 1:
Input: nums = [3,2,1,5,6,4], k = 2
Output: 5

Example 2:
Input: nums = [3,2,3,1,2,4,5,5,6], k = 4
Output: 4

Python Solution


def findKthLargest(nums: List[int], k: int) -> int:
    def quickSelect(left: int, right: int, k: int) -> int:
        pivot = nums[right]
        p = left
        
        # Partition
        for i in range(left, right):
            if nums[i] <= pivot:
                nums[p], nums[i] = nums[i], nums[p]
                p += 1
        nums[p], nums[right] = nums[right], nums[p]
        
        # Count elements greater than pivot
        count = right - p + 1
        
        if count == k:
            return nums[p]
        elif count > k:
            return quickSelect(p + 1, right, k)
        else:
            return quickSelect(left, p - 1, k - count)
    
    return quickSelect(0, len(nums) - 1, k)

Heap Solution:


def findKthLargest(nums: List[int], k: int) -> int:
    # Create min heap
    heap = []
    for num in nums:
        heapq.heappush(heap, num)
        if len(heap) > k:
            heapq.heappop(heap)
    
    return heap[0]

Java Solution


class Solution {
    public int findKthLargest(int[] nums, int k) {
        PriorityQueue minHeap = new PriorityQueue<>();
        
        for (int num : nums) {
            minHeap.offer(num);
            if (minHeap.size() > k) {
                minHeap.poll();
            }
        }
        
        return minHeap.peek();
    }
}

JavaScript Solution


/**
 * @param {number[]} nums
 * @param {number} k
 * @return {number}
 */
var findKthLargest = function(nums, k) {
    const quickSelect = (left, right, k) => {
        const pivot = nums[right];
        let p = left;
        
        // Partition
        for (let i = left; i < right; i++) {
            if (nums[i] <= pivot) {
                [nums[p], nums[i]] = [nums[i], nums[p]];
                p++;
            }
        }
        [nums[p], nums[right]] = [nums[right], nums[p]];
        
        // Count elements greater than pivot
        const count = right - p + 1;
        
        if (count === k) {
            return nums[p];
        } else if (count > k) {
            return quickSelect(p + 1, right, k);
        } else {
            return quickSelect(left, p - 1, k - count);
        }
    };
    
    return quickSelect(0, nums.length - 1, k);
};

C# Solution


public class Solution {
    public int FindKthLargest(int[] nums, int k) {
        var minHeap = new PriorityQueue();
        
        foreach (int num in nums) {
            minHeap.Enqueue(num, num);
            if (minHeap.Count > k) {
                minHeap.Dequeue();
            }
        }
        
        return minHeap.Peek();
    }
}

Complexity Analysis

  • Time Complexity:
    • QuickSelect: O(n) average case, O(nÂ²) worst case
    • Heap: O(n log k)
  • Space Complexity:
    • QuickSelect: O(1)
    • Heap: O(k)

Solution Explanation

There are two main approaches to solve this problem:

  • QuickSelect:
    • Modified quicksort algorithm
    • Only process required partition
    • In-place modification
    • Better average case
  • Heap:
    • Use min heap of size k
    • Keep k largest elements
    • More stable performance
    • Better for streaming data

Key Points

  • Partition strategy
  • Heap maintenance
  • Space-time trade-off
  • In-place vs extra space

Common Pitfalls

  • Pivot selection
  • Heap size control
  • Edge cases
  • Memory management

Edge Cases

  • Empty array
  • Single element
  • Duplicate elements
  • k = 1 or k = length
