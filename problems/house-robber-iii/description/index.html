337. House Robber III

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

The thief has found himself a new place for his thievery again. There is only one entrance to this area, called root.

Besides the root, each house has one and only one parent house. After a tour, the smart thief realized that all houses in this place form a binary tree. It will automatically contact the police if two directly-linked houses were broken into on the same night.

Given the root of the binary tree, return the maximum amount of money the thief can rob without alerting the police.

Example 1:

Input: root = [3,2,3,null,3,null,1]
Output: 7
Explanation: Maximum amount of money the thief can rob = 3 + 3 + 1 = 7

Example 2:

Input: root = [3,4,5,1,3,null,1]
Output: 9
Explanation: Maximum amount of money the thief can rob = 4 + 5 = 9

Constraints:

  • The number of nodes in the tree is in the range [1, 10⁴]
  • 0 <= Node.val <= 10⁴

Solution

Python Solution

class Solution:
    def rob(self, root: Optional[TreeNode]) -> int:
        def dfs(node):
            if not node:
                return (0, 0)  # (rob this node, not rob this node)
            
            left = dfs(node.left)
            right = dfs(node.right)
            
            # If we rob this node, we cannot rob its children
            rob = node.val + left[1] + right[1]
            # If we don't rob this node, we can choose to rob or not rob its children
            not_rob = max(left) + max(right)
            
            return (rob, not_rob)
        
        return max(dfs(root))

Time Complexity: O(n)

Where n is the number of nodes in the binary tree.

Space Complexity: O(h)

Where h is the height of the tree, for the recursion stack.

Java Solution

class Solution {
    public int rob(TreeNode root) {
        int[] result = dfs(root);
        return Math.max(result[0], result[1]);
    }
    
    private int[] dfs(TreeNode node) {
        if (node == null) {
            return new int[]{0, 0};  // {rob this node, not rob this node}
        }
        
        int[] left = dfs(node.left);
        int[] right = dfs(node.right);
        
        // If we rob this node, we cannot rob its children
        int rob = node.val + left[1] + right[1];
        // If we don't rob this node, we can choose to rob or not rob its children
        int not_rob = Math.max(left[0], left[1]) + Math.max(right[0], right[1]);
        
        return new int[]{rob, not_rob};
    }
}

class Solution {
private:
    pair dfs(TreeNode* node) {
        if (!node) {
            return {0, 0};  // {rob this node, not rob this node}
        }
        
        auto left = dfs(node->left);
        auto right = dfs(node->right);
        
        // If we rob this node, we cannot rob its children
        int rob = node->val + left.second + right.second;
        // If we don't rob this node, we can choose to rob or not rob its children
        int not_rob = max(left.first, left.second) + max(right.first, right.second);
        
        return {rob, not_rob};
    }
    
public:
    int rob(TreeNode* root) {
        auto result = dfs(root);
        return max(result.first, result.second);
    }
};

C# Solution

public class Solution {
    public int Rob(TreeNode root) {
        var result = DFS(root);
        return Math.Max(result.Item1, result.Item2);
    }
    
    private (int, int) DFS(TreeNode node) {
        if (node == null) {
            return (0, 0);  // (rob this node, not rob this node)
        }
        
        var left = DFS(node.left);
        var right = DFS(node.right);
        
        // If we rob this node, we cannot rob its children
        int rob = node.val + left.Item2 + right.Item2;
        // If we don't rob this node, we can choose to rob or not rob its children
        int not_rob = Math.Max(left.Item1, left.Item2) + Math.Max(right.Item1, right.Item2);
        
        return (rob, not_rob);
    }
}

Approach Explanation

The solution uses dynamic programming with DFS:

  1. Key Insight:
    • Two states per node
    • Bottom-up approach
    • Optimal substructure
    • No overlapping subproblems
  2. Algorithm Steps:
    • DFS traversal
    • State tracking
    • Decision making
    • Result combination

Example walkthrough:

  • Visit each node
  • Calculate states
  • Make decisions
  • Return maximum

Optimization insights:

  • No memoization needed
  • Constant space per node
  • Single traversal
  • Optimal decisions

Edge Cases:

  • Empty tree
  • Single node
  • All same values
  • Unbalanced tree
