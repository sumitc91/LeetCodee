375. Guess Number Higher or Lower II

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

We are playing a Guess Game. The game is as follows:

I pick a number from 1 to n. You have to guess which number I picked.

Every time you guess wrong, I'll tell you whether the number I picked is higher or lower.

However, when you guess a particular number x, and you guess wrong, you pay $x. You win the game when you guess the number I picked.

Return the minimum amount of money you need to guarantee a win regardless of what number I picked.

Example 1:

Input: n = 10
Output: 16
Explanation: The winning strategy is as follows:
- The range is [1,10]. Guess 7.
    - If this is my number, your total is $0. Otherwise, you pay $7.
    - If my number is higher, the range is [8,10]. Guess 9.
        - If this is my number, your total is $7. Otherwise, you pay $9.
        - If my number is higher, it must be 10. Guess 10. Your total is $7 + $9 = $16.
        - If my number is lower, it must be 8. Guess 8. Your total is $7 + $9 = $16.
    - If my number is lower, the range is [1,6]. Guess 3.
        - If this is my number, your total is $7. Otherwise, you pay $3.
        - If my number is higher, the range is [4,6]. Guess 5.
            - If this is my number, your total is $7 + $3 = $10. Otherwise, you pay $5.
            - If my number is higher, it must be 6. Guess 6. Your total is $7 + $3 + $5 = $15.
            - If my number is lower, it must be 4. Guess 4. Your total is $7 + $3 + $5 = $15.
        - If my number is lower, the range is [1,2]. Guess 1.
            - If this is my number, your total is $7 + $3 = $10.
            - If my number is higher, it must be 2. Guess 2. Your total is $7 + $3 + $1 = $11.

Example 2:

Input: n = 1
Output: 0
Explanation: There is only one possible number, so you can guess 1 and not have to pay anything.

Example 3:

Input: n = 2
Output: 1
Explanation: There are two possible numbers, 1 and 2.
- Guess 1.
    - If this is my number, your total is $0.
    - If my number is higher, it must be 2. Guess 2. Your total is $1.

Constraints:

  • 1 <= n <= 200

Solution

Python Solution

class Solution:
    def getMoneyAmount(self, n: int) -> int:
        # dp[i][j] represents the minimum cost to guarantee a win
        # for the range [i, j]
        dp = [[0] * (n + 1) for _ in range(n + 1)]
        
        # Build the solution bottom-up
        for length in range(2, n + 1):
            for start in range(1, n - length + 2):
                end = start + length - 1
                min_cost = float('inf')
                
                # Try each number as the first guess
                for guess in range(start, end + 1):
                    # Maximum of the cost for the left and right ranges
                    # plus the cost of the current guess
                    cost = guess + max(
                        dp[start][guess - 1] if guess > start else 0,
                        dp[guess + 1][end] if guess < end else 0
                    )
                    min_cost = min(min_cost, cost)
                
                dp[start][end] = min_cost
        
        return dp[1][n]

Time Complexity: O(n³)

Where n is the input number. We have O(n²) states and for each state, we try O(n) guesses.

Space Complexity: O(n²)

For the dynamic programming table.

        // dp[i][j] represents the minimum cost to guarantee a win
        // for the range [i, j]
        int[][] dp = new int[n + 1][n + 1];
        
        // Build the solution bottom-up
        for (int length = 2; length <= n; length++) {

                int end = start + length - 1;

                    );

                }

            }
        }

public class Solution {
    public int GetMoneyAmount(int n) {
        // dp[i,j] represents the minimum cost to guarantee a win
        // for the range [i, j]
        int[,] dp = new int[n + 1, n + 1];
        
        // Build the solution bottom-up
        for (int length = 2; length <= n; length++) {
            for (int start = 1; start <= n - length + 1; start++) {
                int end = start + length - 1;
                int minCost = int.MaxValue;
                
                // Try each number as the first guess
                for (int guess = start; guess <= end; guess++) {
                    // Maximum of the cost for the left and right ranges
                    // plus the cost of the current guess
                    int cost = guess + Math.Max(
                        guess > start ? dp[start, guess - 1] : 0,
                        guess < end ? dp[guess + 1, end] : 0
                    );
                    minCost = Math.Min(minCost, cost);
                }
                
                dp[start, end] = minCost;
            }
        }
        
        return dp[1, n];
    }
}

Approach Explanation

The solution uses dynamic programming to find the minimum cost to guarantee a win:

  1. Key Insights:
    • Minimax strategy for optimal play
    • Dynamic programming for overlapping subproblems
    • Bottom-up approach for efficiency
    • Consider worst-case scenarios
  2. Algorithm Steps:
    • Initialize DP table
    • Process increasing ranges
    • Try each possible first guess
    • Take minimum of maximum costs

Implementation Details:

  • 2D DP table for ranges
  • Bottom-up computation
  • Handle boundary cases
  • Optimal substructure

Optimization Insights:

  • Reuse computed results
  • Minimize state transitions
  • Efficient range processing
  • Space optimization possible

Edge Cases:

  • n = 1 (single number)
  • n = 2 (two numbers)
  • Large values of n
  • Optimal guessing strategy
