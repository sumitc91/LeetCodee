282. Expression Add Operators

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

Given a string num that contains only digits and an integer target, return all possibilities to insert the binary operators '+', '-', and/or '*' between the digits of num so that the resultant expression evaluates to the target value.

Note that operands in the returned expressions should not contain leading zeros.

Example 1:

Input: num = "123", target = 6
Output: ["1*2*3","1+2+3"]
Explanation: Both "1*2*3" and "1+2+3" evaluate to 6.

Example 2:

Input: num = "232", target = 8
Output: ["2*3+2","2+3*2"]
Explanation: Both "2*3+2" and "2+3*2" evaluate to 8.

Example 3:

Input: num = "3456237490", target = 9191
Output: []
Explanation: There are no expressions that can be created from "3456237490" to evaluate to 9191.

Constraints:

  • 1 <= num.length <= 10
  • num consists of only digits
  • -2³¹ <= target <= 2³¹ - 1

Solution

Python Solution

class Solution:
    def addOperators(self, num: str, target: int) -> List[str]:
        def backtrack(expr, start, eval, multed):
            if start == len(num):
                if eval == target:
                    ans.append(expr)
                return
            
            for i in range(start, len(num)):
                # Skip leading zero number
                if i != start and num[start] == '0':
                    break
                    
                # Get current number
                cur = int(num[start:i + 1])
                
                # If it's the first number
                if start == 0:
                    backtrack(expr + str(cur), i + 1, cur, cur)
                else:
                    # Add
                    backtrack(expr + '+' + str(cur), i + 1, eval + cur, cur)
                    # Subtract
                    backtrack(expr + '-' + str(cur), i + 1, eval - cur, -cur)
                    # Multiply
                    backtrack(expr + '*' + str(cur), i + 1, eval - multed + multed * cur, multed * cur)
        
        ans = []
        backtrack("", 0, 0, 0)
        return ans

Time Complexity: O(4^n)

For each position, we have 4 choices (no operator, +, -, *).

Space Complexity: O(n)

The recursion stack can go up to n levels deep.

class Solution {
    List ans = new ArrayList<>();
    String num;
    int target;
    
    public List addOperators(String num, int target) {
        this.num = num;
        this.target = target;
        backtrack("", 0, 0, 0);
        return ans;
    }
    
    private void backtrack(String expr, int start, long eval, long multed) {
        if (start == num.length()) {
            if (eval == target) {
                ans.add(expr);
            }
            return;

        }
    }
}

var addOperators = function(num, target) {
    const ans = [];
    
    function backtrack(expr, start, eval, multed) {
        if (start === num.length) {
            if (eval === target) {
                ans.push(expr);
            }
            return;
        }
        
        for (let i = start; i < num.length; i++) {
            // Skip leading zero number
            if (i !== start && num[start] === '0') {
                break;

            const cur = parseInt(num.slice(start, i + 1));

            if (start === 0) {
                backtrack(expr + cur, i + 1, cur, cur);

                backtrack(expr + '+' + cur, i + 1, eval + cur, cur);
                // Subtract
                backtrack(expr + '-' + cur, i + 1, eval - cur, -cur);
                // Multiply
                backtrack(expr + '*' + cur, i + 1, eval - multed + multed * cur, multed * cur);
            }
        }
    }
    
    backtrack("", 0, 0, 0);
    return ans;
};

Approach Explanation

The solution uses backtracking to generate all possible expressions:

  1. Key Insight:
    • Need to try all possible combinations of operators
    • Handle multiplication precedence
    • Track previous multiplication term
    • Avoid leading zeros
  2. Algorithm Steps:
    • Use backtracking to try all combinations
    • For each position, try all possible numbers
    • Apply each operator and recurse
    • Track evaluation and multiplication term

Example walkthrough for "123" and target 6:

  • Start with "1"
  • Try "1+2" and "1*2"
  • From "1+2", try "1+2+3" (=6)
  • From "1*2", try "1*2*3" (=6)
  • Found two valid expressions

Key insights:

  • Handle multiplication precedence carefully
  • Use long to avoid overflow
  • Skip leading zeros
  • Track previous multiplication term

Edge Cases:

  • Single digit numbers
  • Numbers with leading zeros
  • Large numbers causing overflow
  • No valid expressions
