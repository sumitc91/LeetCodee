220. Contains Duplicate III

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

You are given an integer array nums and two integers indexDiff and valueDiff.

Find a pair of indices (i, j) such that:

  • i != j
  • abs(i - j) <= indexDiff
  • abs(nums[i] - nums[j]) <= valueDiff

Return true if such pair exists or false otherwise.

Example 1:

Input: nums = [1,2,3,1], indexDiff = 3, valueDiff = 0
Output: true
Explanation: We can choose (i, j) = (0, 3).
We satisfy the three conditions:
i != j --> 0 != 3
abs(i - j) <= indexDiff --> abs(0 - 3) <= 3
abs(nums[i] - nums[j]) <= valueDiff --> abs(1 - 1) <= 0

Example 2:

Input: nums = [1,5,9,1,5,9], indexDiff = 2, valueDiff = 3
Output: false
Explanation: No pair satisfies all conditions.

Constraints:

  • 2 <= nums.length <= 10^5
  • -10^9 <= nums[i] <= 10^9
  • 1 <= indexDiff <= nums.length
  • 0 <= valueDiff <= 10^9

Solution

Python Solution

from sortedcontainers import SortedList

class Solution:
    def containsNearbyAlmostDuplicate(self, nums: List[int], indexDiff: int, valueDiff: int) -> bool:
        window = SortedList()
        
        for i, num in enumerate(nums):
            # If window size exceeds indexDiff + 1, remove leftmost element
            if i > indexDiff:
                window.remove(nums[i - indexDiff - 1])
            
            # Find position where num should be inserted
            pos = window.bisect_left(num - valueDiff)
            
            # Check if any number in window satisfies the valueDiff condition
            if pos < len(window) and window[pos] <= num + valueDiff:
                return True
                
            window.add(num)
            
        return False

Time Complexity: O(n log k)

Where n is the length of nums and k is indexDiff.

Space Complexity: O(k)

For the sliding window.

Java Solution

class Solution {
    public boolean containsNearbyAlmostDuplicate(int[] nums, int indexDiff, int valueDiff) {
        if (nums == null || nums.length < 2 || indexDiff < 1 || valueDiff < 0) {
            return false;
        }
        
        TreeSet window = new TreeSet<>();
        
        for (int i = 0; i < nums.length; i++) {
            // Find the smallest number >= nums[i] - valueDiff
            Long ceiling = window.ceiling((long)nums[i] - valueDiff);
            
            // Check if any number in window satisfies the valueDiff condition
            if (ceiling != null && ceiling <= (long)nums[i] + valueDiff) {
                return true;
            }
            
            // Add current number to window
            window.add((long)nums[i]);
            
            // Remove leftmost element if window size exceeds indexDiff
            if (i >= indexDiff) {
                window.remove((long)nums[i - indexDiff]);
            }
        }
        
        return false;
    }
}

                window.erase(nums[i - indexDiff]);

JavaScript Solution

/**
 * @param {number[]} nums
 * @param {number} indexDiff
 * @param {number} valueDiff
 * @return {boolean}
 */
var containsNearbyAlmostDuplicate = function(nums, indexDiff, valueDiff) {
    if (nums.length < 2 || indexDiff < 1 || valueDiff < 0) {
        return false;
    }
    
    // Bucket approach since JavaScript doesn't have TreeSet
    const bucket = new Map();
    const w = valueDiff + 1;
    
    for (let i = 0; i < nums.length; i++) {
        const bucketId = Math.floor(nums[i] / w);
        
        // Check current bucket
        if (bucket.has(bucketId)) {
            return true;
        }
        
        // Check adjacent buckets
        if (bucket.has(bucketId - 1) && Math.abs(nums[i] - bucket.get(bucketId - 1)) < w) {
            return true;
        }
        if (bucket.has(bucketId + 1) && Math.abs(nums[i] - bucket.get(bucketId + 1)) < w) {
            return true;
        }
        
        // Add current number to bucket
        bucket.set(bucketId, nums[i]);
        
        // Remove old bucket
        if (i >= indexDiff) {
            const oldBucketId = Math.floor(nums[i - indexDiff] / w);
            bucket.delete(oldBucketId);
        }
    }
    
    return false;
};

Time Complexity: O(n)

Where n is the length of nums.

Space Complexity: O(k)

For the bucket map.

C# Solution

public class Solution {
    public bool ContainsNearbyAlmostDuplicate(int[] nums, int indexDiff, int valueDiff) {
        if (nums == null || nums.Length < 2 || indexDiff < 1 || valueDiff < 0) {
            return false;
        }
        
        SortedSet window = new SortedSet();
        
        for (int i = 0; i < nums.Length; i++) {
            // Find the smallest number >= nums[i] - valueDiff
            var ceiling = window.GetViewBetween((long)nums[i] - valueDiff, long.MaxValue).Min;
            
            // Check if any number in window satisfies the valueDiff condition
            if (ceiling <= (long)nums[i] + valueDiff) {
                return true;
            }
            
            // Add current number to window
            window.Add(nums[i]);
            
            // Remove leftmost element if window size exceeds indexDiff
            if (i >= indexDiff) {
                window.Remove(nums[i - indexDiff]);
            }
        }
        
        return false;
    }
}

Where n is the length of nums and k is indexDiff.

For the SortedSet window.

Approach Explanation

This problem can be solved using several approaches, but the most efficient ones are:

1. Sliding Window with Ordered Set:

  • Maintain a sliding window of size indexDiff + 1
  • Use an ordered set (TreeSet in Java, SortedList in Python) to store elements
  • For each number, find the closest numbers within valueDiff range
  • Remove elements outside the window

2. Bucket Approach (JavaScript):

  • Use buckets of size valueDiff + 1
  • Numbers in the same or adjacent buckets might satisfy the condition
  • Maintain a sliding window by removing old buckets
  • This approach achieves O(n) time complexity

The choice between approaches depends on the language's available data structures and the specific constraints of the problem. The bucket approach is particularly useful in JavaScript where self-balancing trees are not available.
