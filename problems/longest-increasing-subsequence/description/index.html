300. Longest Increasing Subsequence

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

Given an integer array nums, return the length of the longest strictly increasing subsequence.

A subsequence is a sequence that can be derived from an array by deleting some or no elements without changing the order of the remaining elements. For example, [3,6,2,7] is a subsequence of the array [0,3,1,6,2,2,7].

Example 1:

Input: nums = [10,9,2,5,3,7,101,18]
Output: 4
Explanation: The longest increasing subsequence is [2,3,7,101], therefore the length is 4.

Example 2:

Input: nums = [0,1,0,3,2,3]
Output: 4

Example 3:

Input: nums = [7,7,7,7,7,7,7]
Output: 1

Constraints:

  • 1 <= nums.length <= 2500
  • -10⁴ <= nums[i] <= 10⁴

Follow up:

  • Can you come up with an algorithm that runs in O(n log(n)) time complexity?

Solution

Python Solution

class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        # Initialize tails array
        tails = [0] * len(nums)
        size = 0
        
        # Process each number
        for num in nums:
            # Binary search to find position
            left, right = 0, size
            while left < right:
                mid = (left + right) // 2
                if tails[mid] < num:
                    left = mid + 1
                else:
                    right = mid
            
            # Update tails and size
            tails[left] = num
            size = max(size, left + 1)
        
        return size

Time Complexity: O(n log n)

Binary search for each element.

Space Complexity: O(n)

For storing the tails array.

Java Solution

class Solution {
    public int lengthOfLIS(int[] nums) {
        // Initialize tails array
        int[] tails = new int[nums.length];
        int size = 0;
        
        // Process each number
        for (int num : nums) {
            // Binary search to find position
            int left = 0, right = size;
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] < num) {
                    left = mid + 1;
                } else {
                    right = mid;
                }
            }
            
            // Update tails and size
            tails[left] = num;
            if (left == size) {
                size++;
            }
        }
        
        return size;
    }
}

}

Approach Explanation

The solution uses the patience sorting technique:

  1. Key Insight:
    • Maintain array of smallest tails
    • Binary search for insertion
    • Length of tails is LIS length
    • Optimal substructure property
  2. Algorithm Steps:
    • Initialize tails array
    • Process each number
    • Find insertion position
    • Update tails and size

Example walkthrough:

  • nums = [10,9,2,5,3,7,101,18]
  • tails = [2,3,7,18,...]
  • Each position represents length
  • Final length is 4

Alternative approaches:

  • Dynamic programming O(n²)
  • Segment tree O(n log n)
  • Fenwick tree O(n log n)
  • Recursive with memoization

Edge Cases:

  • Single element
  • All same elements
  • Decreasing sequence
  • Multiple valid LIS
