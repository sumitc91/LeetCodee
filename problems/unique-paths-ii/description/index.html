63. Unique Paths II

Medium

Problem Description

You are given an m x n integer array grid. There is a robot initially located at the top-left corner (i.e., grid[0][0]). The robot tries to move to the bottom-right corner (i.e., grid[m-1][n-1]). The robot can only move either down or right at any point in time.

An obstacle and space are marked as 1 or 0 respectively in grid. A path that the robot takes cannot include any square that is an obstacle.

Return the number of possible unique paths that the robot can take to reach the bottom-right corner.

Examples

Example 1:
Input: obstacleGrid = [[0,0,0],[0,1,0],[0,0,0]]
Output: 2
Explanation: There is one obstacle in the middle of the 3x3 grid above.
There are two ways to reach the bottom-right corner:
1. Right -> Right -> Down -> Down
2. Down -> Down -> Right -> Right

Example 2:
Input: obstacleGrid = [[0,1],[0,0]]
Output: 1

Python Solution


def uniquePathsWithObstacles(obstacleGrid: List[List[int]]) -> int:
    if not obstacleGrid or obstacleGrid[0][0] == 1:
        return 0
        
    m, n = len(obstacleGrid), len(obstacleGrid[0])
    dp = [[0] * n for _ in range(m)]
    
    # Initialize first row and column
    dp[0][0] = 1
    for i in range(1, m):
        dp[i][0] = dp[i-1][0] if obstacleGrid[i][0] == 0 else 0
    for j in range(1, n):
        dp[0][j] = dp[0][j-1] if obstacleGrid[0][j] == 0 else 0
    
    # Fill dp array
    for i in range(1, m):
        for j in range(1, n):
            if obstacleGrid[i][j] == 0:
                dp[i][j] = dp[i-1][j] + dp[i][j-1]
    
    return dp[m-1][n-1]

class Solution {
    public int uniquePathsWithObstacles(int[][] obstacleGrid) {
        if (obstacleGrid == null || obstacleGrid.length == 0 || obstacleGrid[0][0] == 1) {
            return 0;
        }
        
        int m = obstacleGrid.length;
        int n = obstacleGrid[0].length;
        int[][] dp = new int[m][n];
        
        // Initialize first row and column
        dp[0][0] = 1;
        for (int i = 1; i < m; i++) {
            dp[i][0] = obstacleGrid[i][0] == 0 ? dp[i-1][0] : 0;
        }
        for (int j = 1; j < n; j++) {
            dp[0][j] = obstacleGrid[0][j] == 0 ? dp[0][j-1] : 0;
        }
        
        // Fill dp array
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (obstacleGrid[i][j] == 0) {
                    dp[i][j] = dp[i-1][j] + dp[i][j-1];
                }
            }
        }
        
        return dp[m-1][n-1];
    }
}

C# Solution


public class Solution {
    public int UniquePathsWithObstacles(int[][] obstacleGrid) {
        if (obstacleGrid == null || obstacleGrid.Length == 0 || obstacleGrid[0][0] == 1) {
            return 0;
        }
        
        int m = obstacleGrid.Length;
        int n = obstacleGrid[0].Length;
        int[,] dp = new int[m,n];
        
        // Initialize first row and column
        dp[0,0] = 1;
        for (int i = 1; i < m; i++) {
            dp[i,0] = obstacleGrid[i][0] == 0 ? dp[i-1,0] : 0;
        }
        for (int j = 1; j < n; j++) {
            dp[0,j] = obstacleGrid[0][j] == 0 ? dp[0,j-1] : 0;
        }
        
        // Fill dp array
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (obstacleGrid[i][j] == 0) {
                    dp[i,j] = dp[i-1,j] + dp[i,j-1];
                }
            }
        }
        
        return dp[m-1,n-1];
    }
}

Complexity Analysis

  • Time Complexity: O(m × n) to fill the dp array
  • Space Complexity: O(m × n) to store the dp array

Solution Explanation

This solution extends the Unique Paths problem to handle obstacles:

  • Key differences:
    • Check for obstacles before calculating paths
    • Initialize carefully considering obstacles
    • Skip cells with obstacles (value = 1)
  • Algorithm steps:
    • Initialize dp array with zeros
    • Handle first row and column specially
    • For each cell (i,j):
      • If no obstacle: dp[i][j] = dp[i-1][j] + dp[i][j-1]
      • If obstacle: leave as 0

Key points:

  • Handle edge cases (empty grid, obstacle at start)
  • Careful initialization of boundaries
  • Similar to original unique paths
  • Obstacles block all paths through them
