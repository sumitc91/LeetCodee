455. Assign Cookies

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

Assume you are an awesome parent and want to give your children some cookies. But, you should give each child at most one cookie.

Each child i has a greed factor g[i], which is the minimum size of a cookie that the child will be content with; and each cookie j has a size s[j]. If s[j] >= g[i], we can assign the cookie j to the child i, and the child i will be content. Your goal is to maximize the number of your content children and output the maximum number.

Example 1:

Input: g = [1,2,3], s = [1,1]
Output: 1
Explanation: You have 3 children and 2 cookies. The greed factors of 3 children are 1, 2, 3. 
And even though you have 2 cookies, since their size is both 1, you could only make the child whose greed factor is 1 content.
You need to output 1.

Example 2:

Input: g = [1,2], s = [1,2,3]
Output: 2
Explanation: You have 2 children and 3 cookies. The greed factors of 2 children are 1, 2. 
You have 3 cookies and their sizes are big enough to gratify all of the children.

Constraints:

  • 1 <= g.length <= 3 * 10⁴
  • 0 <= s.length <= 3 * 10⁴
  • 1 <= g[i], s[j] <= 2³¹ - 1

Solution

Python Solution

class Solution:
    def findContentChildren(self, g: List[int], s: List[int]) -> int:
        # Sort both arrays
        g.sort()
        s.sort()
        
        content_children = 0
        cookie_index = 0
        
        # Try to assign cookies to children
        for child in g:
            # Find the smallest cookie that can satisfy current child
            while cookie_index < len(s) and s[cookie_index] < child:
                cookie_index += 1
            
            # If we found a suitable cookie
            if cookie_index < len(s):
                content_children += 1
                cookie_index += 1
            else:
                break
        
        return content_children

Time Complexity: O(n log n)

Where n is the maximum length between g and s arrays (due to sorting).

Space Complexity: O(1)

Only constant extra space is used.

class Solution {
    public int findContentChildren(int[] g, int[] s) {
        // Sort both arrays
        Arrays.sort(g);
        Arrays.sort(s);
        
        int contentChildren = 0;
        int cookieIndex = 0;
        
        // Try to assign cookies to children

                cookieIndex++;

                contentChildren++;
                cookieIndex++;
            } else {
                break;
            }

        int contentChildren = 0;

        foreach (int child in g) {

            while (cookieIndex < s.Length && s[cookieIndex] < child) {
                cookieIndex++;
            }

            if (cookieIndex < s.Length) {
                contentChildren++;
                cookieIndex++;
            } else {
                break;
            }

}

Approach Explanation

The solution uses a greedy approach to maximize the number of content children:

  1. Key Insights:
    • Sort both arrays first
    • Assign smallest possible cookie
    • One pass through arrays
    • Early termination
  2. Algorithm Steps:
    • Sort greed factors and cookie sizes
    • Iterate through children
    • Find suitable cookies
    • Track assignments

Implementation Details:

  • Array sorting
  • Two-pointer technique
  • Counter maintenance
  • Loop control

Optimization Insights:

  • Early termination
  • Minimal comparisons
  • In-place sorting
  • No extra space

Edge Cases:

  • Empty cookie array
  • No satisfied children
  • All children satisfied
  • Single element arrays
